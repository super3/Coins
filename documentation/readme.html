Dropdown Menu:

Anatomy:

  <ul class="nav nav-tabs nav-stacked">
      <li><a href="index.html">Index</a></li> <!--default link-->
      <li><a href="about.html">About</a></li> <!--default link-->
      <li>
          <a href="profile" rel="toggle[profile-submenu]">Profile <!--toggle link-->
          <div class="dropdown" id="profile-submenu"> <!--submenu container-->
              <ul> <!--submenu list-->
                  <li><a href="view-profile.html">View Profile</a> <!--submenu link-->
                  <li><a href="edit-profile.html">Edit Profile</a> <!--submenu link-->
              </ul>
          </div>
      </li>
      <li>
        <a href="#" rel="toggle[messages-submenu]">Messages</a>
        <div class="dropdown" id="messages-submenu">
            <ul>
                <li><a href="messages-feed.html">Feed</a></li>
                <li><a href="message.html">Read</a></li>
                <li><a href="compose-message.html">Compose</a></li>
            </ul>
        </div>
      </li>
      ...
      ...
  </ul>

Anatomy explanation:

rel="toggle[profile-submenu]": toggles a any div with id="my-submenu"
id="profile-submenu": identifies the submenu
.dropdown: applies respective CSS styles to submenu

JavaScript:

Every submenu is also referenced in script.js file: 
  // collapsible submenus setup
  animatedcollapse.addDiv('profile-submenu', 'fade=1, group=submenus,'),
  animatedcollapse.addDiv('messages-submenu', 'fade=1, group=submenus, persist=1'),
  animatedcollapse.addDiv('billing-submenu', 'fade=1, group=submenus'),
  animatedcollapse.addDiv('support-submenu', 'fade=1, group=submenus')
  animatedcollapse.ontoggle=function($, divobj, state){
  }
  animatedcollapse.init()

JavaScript explanation:

profile-submenu: ID used in HTML
fade=1: optional fade effect when opening/closing (boolean)
group=submenus: group name can be anything, submenus sharing the same group name are tied together to have only one submenu open at any time
persist=1: (boolean) remembers the submenu state (open or closed) within the browser session. In grouped contents, enabling persist on one submenu automatically enables it on the other submenus.

Tables:

As you might have guessed, tables are not kinda friendly with responsive design, so here's when you can use the .optional class to hide those "not-so-critical" cells from resolutions under 979px. Don't use .desktop-visible because they don't work quite good with data tables. I would only recommend to leave a maximum of 3 columns viewable for mobile devices. If you really want to go all the way through with your talbes I would recommend you to check out these experimental options: CSS Tricks Tables and Zurb Tables.

Login Form and Search Form (Wiki)

For aesthetical purposes any .btn element (link or button) inside the login form and wiki search form will automatically become a full-width block element under 1200px. Take this into account or over-write it in style.css file.

Flot Charts

Use the .chart class to give the chart container a width and height. Add the required JS files to every page where you want to put the charts. Refer to the Flot Chart website to read the full documentation. Don't worry it's pretty easey :) Here's an example: 
  <head>
    <!--[if lt IE 8]>
      <script language="javascript" type="text/javascript" src="js/excanvas.min.js"></script>
    <![endif]-->
  </head>

  <body>
    <div id="chart-1" class="chart">
      <!--this is the container, your chart will be generated here-->
    </div>

    <!--required JS files-->
    <script src="js/jquery.flot.js"></script> <!--flot library-->
    <script src="js/jquery.flot.pie.js"></script> <!--pie plugin (if needed)-->
    <script src="js/jquery.flot.resize.js"></script> <!--resize plugin (if needed)-->
    <script src="js/jquery.flot.demo.js"></script> <!--setup file-->
  </body>
